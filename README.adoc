= Evolution Rules: Filesystem
:toc:
:icons: font

Cross-OS filesystem path equivalency rules (Linux, Windows, macOS, BSD) for the link:https://github.com/AMDphreak/evolution-engine[Evolution Engine].

== Ecosystem

* link:https://github.com/AMDphreak/evolution-engine[Evolution Engine] — The core binary that resolves and applies these rules.
* link:https://github.com/AMDphreak/evolution-engine-action[Evolution Engine Action] — Modular GitHub Action to automate code evolution in CI pipelines.
* link:https://github.com/AMDphreak/evolution-rules-code[Evolution Rules: Code] — Rules for migrating source code between library versions.
* link:https://github.com/AMDphreak/evolution-rules-filesystem[Evolution Rules: Filesystem] — This repository.

== Overview

This repository is a **Content Repository** for the `filesystem` domain. It provides abstract "intents" (e.g., `lib_dir`, `config_dir`) and maps them to literal paths across different operating systems and distributions.

== Directory Structure

[source]
----
rules/
├── linux/              <-- OS Family
│   └── ubuntu/         <-- Distribution
│       └── 22.04/      <-- Version
├── windows/
└── mac/
----

== Usage with Evolution Engine

To resolve a filesystem intent across platforms:

[source,bash]
----
evolution-engine --domain filesystem \
  --rules-repo https://github.com/AMDphreak/evolution-rules-filesystem \
  --to linux/ubuntu/22.04 lib_dir
----

== Contribution

Add new path mappings to the `rules/<os>/<distro>/<version>/` folders. Use the intent name as the filename (e.g., `rules/linux/ubuntu/lib_dir.sdl`).
